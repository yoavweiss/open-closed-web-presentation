The open-closed principle of the Web

The open Web

History

  • Invented in 1991 by Tim Berners-Lee in CERN
  • Became popular in 1995 with Netscape Navigator

Principles

  • No patents
  • Text based
  • Open specs
  • No proprietary tech

Specified by

  • W3C
  • WHATWG
  • ECMA

Technologies

  • HTML
  • CSS
  • JavaScript
  • SVG
  • XML
  • Others...

What's not Web?

  • Anything that requires a browser plugin - Flash, Java, Silverlight, ActiveX
  • Native apps - iOS, Android
  • Anything that works only in one browser/rendering engine

What's so good about it?

  • Hackable - "view source"
  • Open culture - open source {tools, standards, browsers}
  • Amazing cross-platform reach

So what's the closed part?

Your private data

(should be)

Privacy threats

  • Private data sold to 3rd parties
  • Browsing data collection
  • Security vulebrabilities
  • Phishing

Private data on social networks

Just don't!

Browsing data collection

  • 3rd party cookies
  • Analytics scripts
  • Social widgets
  • Search engine redirections
  • URL shorteners
  • Link "source" in shared URLs
  • Referrer
  • Toolbars

What can users do about that?

  • Set browser to block 3rd party cookies
  • Block analytics/data collection domains
  • Don't install browser Toolbars. Don't!

Security vulnerabilities

  • Cross-site scripting
  • Click-jacking
  • Session cookies theft
  • Password leaks

Cross-site scripting

  • JS code uploaded by one user, ran on another user's session
  • Enables data theft
  • Can be site related, or self inflicted

What can users do about that?

  • Don't copy/paste data URLs
  • Be aware the problem exists
  • Prefer security conscious Web services

Click-jacking

  • Logged in legitimate site loaded as transparent iframe
  • Clicks on malicious site propagated as clicks on legitimate site
  • Modern browsers define clickjacking protection headers

What can users do about that?

  • Use "incognito"/"private" mode when going to suspicious sites
  • Use a modern browser
  • Prefer security conscious Web services

Session cookie theft

  • Sites maintain logged in users through session cookies
  • Logged in user browses on an unsecured network
  • Malicious user listening on the network can steal cookie and log in as user

What can users do about that?

  • Prefer HTTPS over HTTP
  • Avoid unsecured networks if possible
  • HTTP + unsecured network == PWNed

Password leaks

  • Sites maintain user passwords in clear text/unsecured hash
  • Sites are hacked
  • Leaked password can be used for other services

What can users do about that?

  • Avoid using the same password on multiple services
  • Avoid services that send you the password in clear-text
  • Prefer OpenID/OAuth/Persona authentication over user/pwd
  • Prefer security conscious Web services

Phishing

  • User clicks on link that appears to be legitimate
  • Link brings user to a copy of legitimate site
  • User is convinced to type in private information (passwords, credit card, etc)

What can users do about that?

  • Don't click links in emails. Type URLs yourselves
  • HTTPS for private information. Check the certificate

In any case...

often humans are the weakest link

Conclusions

  • The Web is good
  • Users must be privacy/security conscious
